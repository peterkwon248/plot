# datda (닫다) - 컨셉 문서

## 1. 정체성

**한 줄 정의:** 느리지만 분명한 성취 도구

**포지셔닝:** 할 일 앱이 아니라 종료 장치.
매일 하나를 닫는 것으로, 오늘을 살았다는 감각을 준다.

**타겟 사용자:** 큰 목표 앞에서 막히는 사람.
무엇부터 해야 할지 모르고, 전체를 보면 부담을 느끼는 사람.

**핵심 감정:**
> "오늘도 하나를 닫았다. 오늘을 헛되이 보내지 않았다."

**경쟁 앱과의 차이:**
| 일반 할 일 앱 | datda |
|--------------|-------|
| "이것도 해, 저것도 해" | "오늘은 이것 하나만" |
| 목록이 쌓이면 부담 | 항상 하나만 보임 |
| 더 많이, 더 빨리 | 적게, 천천히, 확실하게 |
| 완료 체크의 쾌감 | 닫힘의 안도감 |

---

## 2. 핵심 원칙

### 원칙 1: 하나만 보여준다
사용자에게는 오늘 닫을 것 하나만 보인다.
프로젝트 전체 로드맵, 남은 단계 수, 예상 완료일 따위는 보여주지 않는다.

### 원칙 2: 전체를 숨긴다
AI가 뒤에서 10~15단계를 쪼개놨어도, 사용자 눈에는 오늘 것만 보인다.
서랍함에 넣어두고, 꺼내는 건 앱이 한다.

### 원칙 3: 재촉하지 않는다
알림 없음. 연속 기록 강조 없음. 빈 날이 있어도 괜찮다.
돌아왔을 때 "오늘 닫을 것"이 조용히 기다리고 있을 뿐.

### 원칙 4: 조용히 기록한다
"닫힘." 한 마디. 과한 칭찬 없음. 통계/그래프/랭킹 없음.
닫힌 것들이 쌓이면, 그 자체가 기록이다.

### 원칙 5: AI는 뒤에서 일한다
AI는 입구(목표 쪼개기)에서만 개입한다.
OMCL 세션 흐름은 코드로 고정. 세션 중에 AI가 끼어들지 않는다.
사용자에게는 결과만 전달한다.

## 3. 화면별 UX 시나리오

### 3-1. 홈 화면

**사용자가 보는 것:**
```
오늘 닫을 것
──────────────────
포트폴리오 메인 페이지 HTML 코딩하기
30분 · 완성

[닫기 시작]

지난 닫힘: 와이어프레임 그리기 · 어제
```

**사용자가 느끼는 것:**
"아, 오늘은 이거 하나만 하면 되는구나."
부담 없음. 선택지 없음. 하나만 있음.

**사용자가 하는 것:**
[닫기 시작] 탭 → OMCL 세션 진입

**규칙:**
- 프로젝트 이름, 전체 로드맵, 진행률 표시 안 함
- 오늘 것 하나만 보임
- 프로젝트가 없으면: "서랍함에 목표를 넣어보세요"
- 하단에 지난 닫힘 1개만 살짝 표시

---

### 3-2. OMCL 세션 (OPEN -> MAKE -> CLOSE -> LOCK -> END)

**흐름:**
- OPEN: 오늘 닫을 것 확인 (홈에서 넘어온 값이 미리 채워져 있음)
- MAKE: 타이머와 함께 집중
- CLOSE: 종료 선언
- LOCK: 작업 잠금
- END: 간단한 회고 (상태, 다음 행동)

**변경 없음.** 현재 흐름이 이미 잘 되어있음.
OPEN에서 직접 입력 대신, 추천값이 채워진 상태로 시작.

---

### 3-3. Final 화면 (세션 완료)

**사용자가 보는 것:**
```
닫힘.

(잠시 후)
포트폴리오 만들기 — 4번째 닫힘

[홈으로]
```

**사용자가 느끼는 것:**
"오늘도 했다. 하나를 닫았다."
프로젝트에서 몇 번째인지 살짝 알려줌 → 쌓이고 있다는 감각.

**규칙:**
- 남은 단계 수 안 보여줌 ("6개 남았습니다" X)
- "N번째 닫힘"만 표시 → 성취가 쌓이는 느낌

---

### 3-4. 서랍함

**사용자가 보는 것 (접었을 때):**
```
서랍함
1일 1회만 열 수 있습니다.
──────────────────
📁 포트폴리오 만들기      4/10
📁 블로그 시작하기        1/7

[새 목표 넣기]
```

**사용자가 보는 것 (폴더를 열면):**
```
📁 포트폴리오 만들기
   1단계 ✓ (10/10 완료)
   2단계 ▶ (3/8 진행 중)

   ✓ 레퍼런스 수집
   ✓ 자기소개 작성
   ✓ 와이어프레임
   ▶ HTML 코딩        ← 오늘
   · 스타일링
   · 반응형
   ...
```

**규칙:**
- 1일 1회 접근 제한 유지 ("자꾸 들여다보지 마")
- 여기서는 확인만. 세션 시작 안 함.
- 전체 로드맵을 볼 수 있지만, 평소에는 접혀있음.

---

### 3-5. AI 쪼개기 (Chat)

**사용자가 보는 것:**
```
큰 목표를 말해주세요.

사용자: "포트폴리오 사이트 만들고 싶어"

AI: "10단계로 쪼개서 서랍함에 넣었습니다.
     내일부터 하루에 하나씩 안내해드릴게요."

[서랍함 보기]
```

**사용자가 느끼는 것:**
"이렇게 큰 걸 쪼개주는구나. 나는 오늘 것만 하면 되겠다."

**규칙:**
- AI가 쪼갠 단계 목록을 하나하나 보여주지 않음
- "N단계로 쪼갰습니다" 한 줄이면 충분
- 궁금하면 서랍함에서 확인
- 서랍함에 자동 저장

---

### 3-6. 닫힌 것들 (히스토리)

**사용자가 보는 것:**
```
닫힌 것들
──────────
포트폴리오 HTML 코딩하기     오늘
와이어프레임 그리기          어제
자기소개 초안 작성           2일 전
레퍼런스 수집               3일 전
```

**사용자가 느끼는 것:**
"이만큼 해왔구나."

**규칙:**
- 통계 없음. 그래프 없음. 연속일수 강조 없음.
- 그냥 닫힌 것들의 흔적. 내가 살아온 날들.

## 4. 프로젝트 생명주기

### 생성
```
사용자: "포트폴리오 만들고 싶어" (AI 쪼개기 또는 서랍함에서 직접 입력)
    ↓
AI가 뒤에서 10단계로 분해
    ↓
서랍함에 프로젝트로 저장
    ↓
내일부터 홈에 "오늘 닫을 것" 자동 표시
```

### 진행
```
매일 홈 → "오늘 닫을 것" 확인 → OMCL 세션 → 닫힘
    ↓
다음 날 → 다음 단계가 자동으로 올라옴
    ↓
(반복)
```

### 완료
```
📁 포트폴리오 만들기  10/10 완료
    ↓
"모든 단계를 닫았습니다."
    ↓
[새 프로젝트 시작]  [이 프로젝트 심화]
```

### 심화 (회차 시스템)
```
[이 프로젝트 심화] 선택
    ↓
AI가 이전 회차 맥락을 참고하여 다음 레벨 생성
"포트폴리오 만들기 · 2단계"
    ↓
더 높은 난이도의 새 로드맵
```

**회차별 성격:**
| 단계 | 성격 | 예시 |
|------|------|------|
| 1단계 | 기초. "일단 해내기" | HTML로 기본 구조 만들기 |
| 2단계 | 심화. "더 잘 해보기" | 반응형, 다크모드, 애니메이션 |
| 3단계 | 숙련. "제대로 만들기" | SEO, 성능 최적화, 배포 |

## 5. 기술 스택

- **프론트엔드:** Next.js (App Router) + TypeScript + Tailwind CSS
- **상태관리:** Zustand (persist → LocalStorage)
- **AI:** Gemini 2.0 Flash (무료 티어)
- **애니메이션:** Framer Motion
- **배포:** Vercel
- **인증:** 없음 (로컬 저장)
- **폰트:** Pretendard

## 6. 톤 & 무드

- **색상:** 다크. 눈이 편한 정도. (#0a0a0b 배경, #a78bfa 악센트)
- **텍스트:** 짧고 담백. 느낌표 없음. 이모지 없음.
- **전환:** 느리고 부드럽게. 급하지 않게.
- **소리:** 없음.
- **알림:** 없음.

**좋은 예:**
- "닫힘."
- "오늘 닫을 것"
- "서랍함에 넣었습니다."

**나쁜 예:**
- "축하합니다! 오늘도 목표를 달성했어요!"
- "3일 연속 달성! 스트릭을 유지하세요!"
- "오늘 할 일이 5개 남았습니다."

## 7. 디자인 원칙: 감성 시각화

**데이터 시각화는 NO, 감성 시각화는 YES.**

그래프, 차트, 통계는 비교와 측정의 도구다.
비교는 압박을 낳고, 압박은 datda의 적이다.

대신 **분위기를 만드는 그래픽**을 사용한다.

### 쓰지 않는 것
| 요소 | 이유 |
|------|------|
| 진행률 그래프/차트 | "더 해야 해" 압박감 |
| 캘린더 히트맵 | 빈 날이 불안해짐. 스트릭 강박 |
| 주간/월간 통계 | 생산성 앱이 되어버림 |
| 배지/레벨/포인트 | 게이미피케이션은 datda가 아님 |

### 쓰는 것
| 요소 | 역할 |
|------|------|
| 카드 한 장 | "오늘 닫을 것"을 물리적으로 느끼게 |
| 부드러운 전환 | 급하지 않은 리듬 |
| 서랍 인터랙션 | 열고 닫는 촉감 |
| 담백한 숫자 (4/10) | 정보 최소화. 작고 조용하게 |
| 여백 | 비어있는 것이 곧 편안함 |

### 디자인 톤
- 종이 위에 연필로 쓴 듯한 가벼움
- 어둡지만 무겁지 않은 배경
- 색은 악센트 하나 (#a78bfa)만 절제해서 사용
- 요소 간 충분한 여백 → 숨 쉴 공간
